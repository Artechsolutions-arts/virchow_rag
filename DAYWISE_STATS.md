## Worker Kill Cycle Details

Each worker loads the DotsOCR model (~24 GB on start) then grows RSS by ~0.3–0.5 GB per 5 min.  
When any worker hits **62 GB RSS**, the kill/restart sequence runs:

1. SIGKILL the worker
2. Reset orphaned `processing` docs → `pending` in DB  
3. Start fresh replacement worker (loads model in ~5–10 min)  
4. Re-queue all pending docs to RabbitMQ

**21 kill cycles** have run across the full pipeline. Kill Cycle 21 (2026-05-13 12:10) was triggered by all 3 workers entering `Us` (uninterruptible sleep) after an Ollama model-unload event — all 3 were killed simultaneously and replaced. Each cycle adds ~10–15 min downtime to the affected worker while the other 2 continue uninterrupted.
